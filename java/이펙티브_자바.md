### GC

**개념:**

- Mark, Sweep, Compact
  - Mark: 사용 중인 객체를 표시
  - Sweep: 사용 중이지 않은 객체를 제거
- Comapact: 메모리 단편화를 방지하기 위해 사용 중인 객체를 압축
- Young Generation (Eden, Survivor 1, Survivor 2), Old Generation
  - Young Generation: 새로 생성된 객체가 저장되는 공간
    - Eden: 새로 생성된 객체가 저장되는 공간
    - Survivor 1, Survivor 2: Eden에서 살아남은 객체가 저장되는 공간
  - Old Generation: Young Generation에서 살아남은 객체가 저장되는 공간
- Minor GC: Young Generation에서 발생하는 GC
- Major GC: Old Generation에서 발생하는 GC
- Full GC: Young Generation과 Old Generation 모두에서 발생하는 GC
- Throughput, **Latency (Stop-The-World)**, Footprint
  - Throughput: GC가 얼마나 효율적으로 메모리를 관리하는지를 나타내는 지표
  - Stop-The-World: GC가 실행되는 동안 애플리케이션이 멈추는 현상
  - Footprint: GC가 사용하는 메모리의 양
- Serial, Parallel, CMS, G1, ZGC, Shenandoah
  - Serial: 단일 스레드로 GC를 수행
  - Parallel: 여러 스레드를 사용하여 GC를 수행
  - CMS: Concurrent Mark-Sweep, 멀티 스레드로 GC를 수행
  - G1: Garbage First, 메모리를 여러 개의 영역으로 나누어 GC를 수행
  - **ZGC**: Low Latency GC, 짧은 지연 시간으로 GC를 수행
  - Shenandoah: Low Pause Time GC, 짧은 정지 시간을 목표로 하는 GC
